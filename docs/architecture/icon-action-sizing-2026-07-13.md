# アイコン操作のサイズ設計（2026-07-13）

## 背景

イベント履歴のお気に入りStarは、操作領域に対して記号が小さく、未選択時には操作可能な場所だと認識しづらかった。また、作成フォームのヘルプ操作は幅20pxに固定された文字の`?`で、タッチ操作とキーボード操作の基準を満たしていなかった。

## 設計判断

- アイコン単独ボタンはデスクトップ40px、`pointer: coarse`では44pxの正方形を共通基準とする。
- アイコン単独操作の記号は原則18〜20pxとし、Starは形状を認識しやすい20px・`strokeWidth=2.25`とする。
- お気に入りの未選択状態にも中立色の境界線と背景を表示し、選択状態はStarの塗り、背景色、`aria-pressed`を併用して色だけに依存しない。
- フォームの補足説明にはLucideの`CircleHelp`を使用し、文字記号をアイコンの代用にしない。
- 補足説明ボタンを通常のタブ順へ含め、ラベルとフォーカスリングでキーボード操作を保証する。
- テキスト付きの補助ボタンは、記号が補助的である限り16pxを下限として扱う。

## 影響範囲

共通の`.btn-icon`、イベント履歴、お気に入り操作、作成フォームの期間・時間帯・時間枠ヘルプ、ヘッダーのテーマ切り替えと認証操作へ適用する。URL、データモデル、Server Actions、認証方式、業務ルールは変更しない。
