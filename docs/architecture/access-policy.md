# アクセス権限・閲覧方針

作成日: 2026-01-25
作成者: Codex（GPT-5）

## 背景

- 管理者（Admin）概念を廃止し、全ユーザーを同一の権限として扱う方針に変更した。
- 参加者一覧・回答は「リンクを知っている人なら誰でも閲覧可能」とする運用を採用する。
- 既存DBの変更は最小限にし、既存の `admin_token` は生成・保存のみ継続する。

## 方針（結論）

- **権限の区別は行わない。** 主催者/参加者のUI上の区分は行うが、操作権限は同一とする。
- **参加者一覧・回答は誰でも閲覧可能** とする（公開リンクにアクセスできるユーザーを前提）。
- **`admin_token` は将来拡張用の予約領域として保持**し、現在は利用しない。

## 影響範囲

- イベント確定などの操作は「権限チェック無し」で実行される前提となる。
- RLSは「匿名ユーザーによるSELECTを許可」する現行方針を継続する。
- DBスキーマの変更は行わず、`admin_token` カラムは維持する。

## セキュリティ・運用上の注意

- 公開リンクの取り扱いが事実上のアクセス制御となる。
- 公開リンクの共有範囲を明示し、必要に応じて告知文言で注意喚起する。
- 将来的に権限分離を再導入する場合は、`admin_token` を起点に再設計する。
