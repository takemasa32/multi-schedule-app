# デザイン改善メモ（2026-05-11）

## 背景

LP と主要 UI の見た目が、バッジ・比較表・装飾的なグラデーションに寄り、DaySynth の価値である「参加者の予定を重ねて、最適日を決める」体験がやや伝わりにくくなっていた。

## 方針

- LP は説明量を減らし、候補作成・共有・確定の流れを最短で理解できる構成にする。
- 装飾は生成画像のヒーローイメージに絞り、背景の円形 SVG や過度なバッジを撤去する。
- 生成画像は `public/images/landing/` に配置し、DaySynth の「重ね合わせ」コンセプトを表す補助ビジュアルとして扱う。画像内には不正確なロゴ・アイコン・文字を含めない。
- 本番参照するヒーロー画像は WebP に最適化し、元生成 PNG と `tmp/imagegen/` の作業ファイルは保持しない。
- ヒーロー画像には独自アニメーションを付けず、主要な読み取りと操作導線を優先する。
- スマホ幅ではヒーロー画像を縦積みにせず、テキスト右奥へ薄く重ねることでファーストビューの縦長化を抑える。PC幅では左テキスト・右画像の2カラムを維持する。
- ログイン導線は必須感を出さず、履歴・お気に入り・自分の予定活用など、繰り返し使う場合の追加価値として案内する。
- LP の静的な本文はサーバーコンポーネントとして描画し、履歴・お気に入り表示だけをクライアントコンポーネントへ分離する。
- 実利用画面は見出し・カード・入力方式選択の視覚的圧を下げ、フォームの進行と次アクションが読み取りやすい状態を優先する。
- OG画像と共有文言も「予定の重なりを合成する」価値に合わせ、DaySynth の表記ゆれや旧来の「みんなの予定を簡単調整」中心の訴求を整理する。
- OG画像は各SNSやメッセージアプリで縮小・トリミングされる前提で、重要な文字を中央安全域に置き、テキストは2〜3行以内、抽象ビジュアルは細部を減らしてサムネイル時の可読性を優先する。
- OG画像内の略字表現は避け、DaySynth のサービス名と既存アイコンロゴを使う。ホーム用OGではサービス名を最大の視覚要素にし、イベント用OGではイベント名を最大要素にする。
- イベント名の自動改行は CSS 任せにせず、`Intl.Segmenter` による日本語の語分割を起点にする。語途中の分割、禁則対象文字の前後、短い孤立行を避け、`の日程調整` などの定型末尾は意味単位で優先分割する。
- イベント用OGではフォントサイズを下げる前に、右側ビジュアルを背面へ回してタイトル領域を広げる。英数字・記号・絵文字を含む実タイトルでは見た目幅で行数を決め、OG画像内で□表示になりやすい絵文字は除去して空なら「イベント」にフォールバックする。

## 今後の確認観点

- 生成画像内の細部がモバイルで主情報を邪魔していないか。
- 生成画像にロゴ、文字、独自アイコンが混入していないか。
- ライト / ダークそれぞれで背景透過画像に切り替わり、ページ背景になじむか。
- 回答画面・確定画面・アカウント画面で、今回のカード半径と影の控えめな方針に反する箇所が残っていないか。
- 実ユーザーの利用ログに応じて、LP の説明量をさらに減らせるか。
