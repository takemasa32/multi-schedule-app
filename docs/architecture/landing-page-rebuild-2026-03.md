# LP再構築メモ（2026-03）

## 背景

- 既存LPは情報量は十分だった一方で、価値訴求が分散し、初回訪問ユーザーが「何を・どれくらい早く解決できるか」を直感しづらかった。
- Motion（motion.dev）移行後の活用範囲をLPに広げ、視覚的な説得力と体験価値を強化する必要があった。

## 目的

1. ファーストビューで「日程調整が速く終わる」価値を明確化する。
2. スマホ/PCの両方で読みやすく、操作しやすいレイアウトへ統一する。
3. サーバー責務（metadata）とクライアント責務（アニメーション描画）を分離する。

## 実装方針

- `src/app/page.tsx` はサーバーコンポーネントとして metadata のみ担当。
- LP本文は `src/components/landing/landing-page-client.tsx` に集約し、Motionによる表示演出を実装。
- アニメーションは `useReducedMotion` で低モーション設定に追従。
- ヒーローに SVG ヒートマップの簡易アニメーションを配置し、「回答が集まり最適日が見つかる」体験を視覚化。
- フォントは `next/font/google` で `Noto Sans JP` + `Inter` を導入し、日本語/英数の可読性を改善。

## 主要変更点

- ヒーローを再設計し、CTA・導線・主要指標を同一視野で確認できる構成へ変更。
- 課題訴求 → 機能訴求 → 3ステップ導線 → 履歴確認 → 最終CTA の順でストーリー化。
- 既存の `hero-section.tsx` は役割重複を解消するため廃止。

## 影響範囲

- トップページ（`/`）の表示構成とアニメーション挙動。
- 全体フォント指定（`src/app/layout.tsx`, `src/app/globals.css`）。

## 今後の拡張案

- 実利用データ（離脱率/スクロール率）を計測し、セクション順序やCTA文言をA/Bテストで最適化。
- SVGヒートマップを実データ風モックへ発展させ、ユースケース別に切り替え可能にする。
