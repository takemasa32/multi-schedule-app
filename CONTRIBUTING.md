# Contributing to DaySynth

DaySynth への貢献ありがとうございます。公開 OSS として安全に運用するため、変更は小さく、意図が追いやすい形で進めてください。

## 進め方

1. 変更前に `README.md` と `specifications/`、関連する設計メモを確認してください。
2. 影響範囲が広い変更は、先に issue で目的と方針を共有してください。
3. 実装は最小スライスから始め、既存の挙動を壊さないことを優先してください。
4. 変更後は `npm run lint`、`npm run typecheck`、`npm run test:ci` を実行してください。
5. UI やフロー変更には、必要に応じて Playwright の E2E テストを追加してください。

## コーディング規約

- コメント、コミットメッセージ、PR 説明は日本語で書いてください。
- 秘密情報、個人情報、非公開 URL はコミットしないでください。
- 不要なコードやコメントは残さないでください。
- 仕様外の変更を行う場合は、関連ドキュメントも更新してください。

## PR の出し方

- 変更の目的
- 主な変更点
- 実行した確認コマンド
- 影響範囲
- 必要ならスクリーンショット

## 相談先

不明点がある場合は、実装前に issue か PR コメントで確認してください。推測で進めるより、前提を明確にしてから着手するほうが安全です。
